# 🚦 Traffic Management System - Vercel Deployment Guide\n\n## Overview\nThis is a serverless version of the Traffic Management System optimized for Vercel deployment.\n\n### Changes from Original\n✅ Serverless architecture (Flask on Vercel Python runtime)  \n✅ Frame-based detection (instead of video streaming)  \n✅ Lightweight YOLOv8n model (140MB vs 500MB+)  \n✅ Stateless API design  \n✅ No video streaming (use image uploads instead)  \n\n---\n\n## 📋 Prerequisites\n\n1. **Vercel Account** - Sign up at https://vercel.com\n2. **Git Repository** - Push your code to GitHub\n3. **Python 3.11+** (for local testing)\n4. **Vercel CLI** - Optional but recommended\n\n---\n\n## 🚀 Deployment Steps\n\n### Step 1: Prepare Your Repository\n\n```bash\n# Initialize git if not already done\ngit init\ngit add .\ngit commit -m \"Ready for Vercel deployment\"\n```\n\n### Step 2: Create .gitignore\n\nCreate `.gitignore` to exclude large files:\n\n```\n__pycache__/\n.pyc\n*.pt\n.env\nvenv/\nnode_modules/\n.DS_Store\n```\n\n### Step 3: Add Environment Variables\n\nCreate `.env.local` for local testing:\n\n```\nADMIN_USER=admin\nADMIN_PASS=admin123\nSECRET_KEY=your-secret-key-here\n```\n\n### Step 4: Push to GitHub\n\n```bash\ngit remote add origin https://github.com/YOUR_USERNAME/REPO_NAME.git\ngit branch -M main\ngit push -u origin main\n```\n\n### Step 5: Deploy on Vercel\n\n#### Option A: Via Vercel Web UI (Easiest)\n\n1. Go to https://vercel.com/new\n2. Select \"Import Git Repository\"\n3. Paste your GitHub repo URL\n4. Click \"Import\"\n5. Add environment variables:\n   - `ADMIN_USER` = `admin`\n   - `ADMIN_PASS` = `admin123`\n   - `SECRET_KEY` = Generate a random string\n6. Click \"Deploy\"\n\n#### Option B: Via Vercel CLI\n\n```bash\n# Install Vercel CLI\nnpm i -g vercel\n\n# Deploy\nvercel\n\n# Follow prompts and add environment variables\n```\n\n---\n\n## 📝 Project Structure (Vercel)\n\n```\ntraffic-management-system/\n├── api/\n│   └── index.py           # Main Flask app\n├── public/\n│   ├── index.html         # Dashboard\n│   └── login.html         # Login page\n├── vercel.json            # Vercel config\n├── requirements.txt       # Python dependencies\n└── .gitignore\n```\n\n---\n\n## 🔑 API Endpoints\n\n### Authentication\n```\nPOST /login\nGET /logout\n```\n\n### Traffic Management\n```\nGET /api/stats              # Get current traffic state\nPOST /api/detect            # Process frame (multipart/form-data)\nPOST /api/service/<lane>    # Toggle service mode\nPOST /api/accident          # Toggle emergency mode\nPOST /api/signal/<lane>/<signal>  # Manually set signal\nGET /api/health            # Health check\n```\n\n---\n\n## 🖼️ Usage\n\n### Login\n- **Username**: `admin`\n- **Password**: `admin123`\n\n### Dashboard\n- **View Stats**: Real-time vehicle counts and PCU values\n- **Upload Frame**: Process traffic images for vehicle detection\n- **Controls**: Toggle service mode and emergency stop\n\n### Frame Processing\n1. Go to \"Process Frame\" tab\n2. Click \"Select Frame\" and choose a traffic image\n3. System analyzes vehicle types and PCU value\n4. Results update dashboard\n\n---\n\n## ⚠️ Limitations on Vercel\n\n| Feature | Original | Vercel |\n|---------|----------|--------|\n| Video Streaming | 24/7 | ❌ Not available |\n| Real-time Processing | Continuous | Frame-based |\n| Execution Time | Unlimited | 12 seconds max |\n| Model Size | 500MB+ | 140MB (YOLOv8n) |\n| Database | In-memory | State per request |\n| Concurrent Users | 10+ | Limited |\n\n---\n\n## 🔧 Configuration\n\nEdit these in `api/index.py`:\n\n```python\nADMIN_USER = \"admin\"       # Change username\nADMIN_PASS = \"admin123\"    # Change password\n```\n\n---\n\n## 🐛 Troubleshooting\n\n### Issue: Deployment fails with \"Model too large\"\n**Solution**: YOLOv8n is already optimized. Ensure `.gitignore` excludes `.pt` files.\n\n### Issue: Login not working\n**Solution**: Check environment variables are set correctly in Vercel dashboard.\n\n### Issue: Frame detection timeout\n**Solution**: Keep images under 5MB and use JPEG format.\n\n### Issue: \"Static files not found\"\n**Solution**: Ensure `public/` folder exists with `index.html`.\n\n---\n\n## 📊 Monitoring\n\nAfter deployment:\n\n1. Go to Vercel Dashboard → Your Project\n2. Navigate to \"Analytics\" tab\n3. Check \"Serverless Functions\" for errors\n4. View \"Logs\" for real-time output\n\n---\n\n## 🔗 Alternative Platforms\n\nIf Vercel doesn't meet your needs, consider:\n\n- **Railway.app** - Full Docker support, persistent storage, 60s timeout\n- **Render.com** - Similar to Railway, free tier available\n- **AWS Lambda** - More complex but scalable\n- **Heroku** - Simple but paid-only now\n\n---\n\n## 📞 Support\n\nFor issues:\n1.
