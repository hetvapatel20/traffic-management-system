# 🚀 Quick Start - Vercel Deployment\n\n## What Changed?\n\nYour app is now **serverless and ready for Vercel**! Here's what was updated:\n\n✅ **New folder structure**:\n```\napi/              → Serverless Python functions\npublic/           → Static frontend files\nvercel.json       → Deployment configuration\n.gitignore        → Ignores large files\n```\n\n✅ **Key improvements**:\n- No video streaming (use image frame uploads instead)\n- YOLOv8n model (140MB) instead of yolov8s/x\n- Stateless API design\n- 12-second execution timeout\n- Automatic model downloading on first deployment\n\n---\n\n## 📖 Step-by-Step Deployment\n\n### 1. Test Locally (Optional)\n```bash\npip install -r requirements.txt\npython test_local.py\n# Go to http://localhost:5000\n```\n\n### 2. Push to GitHub\n```bash\ngit add .\ngit commit -m \"Prepare for Vercel deployment\"\ngit push origin main\n```\n\n### 3. Deploy to Vercel\n\n**Option A - Easy (Web UI):**\n1. Go to https://vercel.com/new\n2. Click \"Import Git Repository\"\n3. Paste your GitHub repo URL\n4. Select **Framework**: Python\n5. Set **Environment Variables**:\n   - `ADMIN_USER` = `admin`\n   - `ADMIN_PASS` = `admin123`\n6. Click \"Deploy\"\n\n**Option B - CLI:**\n```bash\nnpm install -g vercel\nvercel --prod\n# Add environment variables when prompted\n```\n\n---\n\n## 🎯 How to Use After Deployment\n\n### Login\n- Go to your Vercel app URL\n- Username: `admin`\n- Password: `admin123`\n\n### Dashboard\n- **Real-time Stats** - Vehicle counts per lane\n- **Upload Frames** - Process traffic images\n- **Controls** - Toggle service mode & emergency stop\n\n### Process Traffic Frame\n1. Go to \"Process Frame\" tab\n2. Upload a traffic image (.jpg/.png)\n3. System detects vehicles: cars, bikes, buses, trucks\n4. See PCU (Passenger Car Unit) value\n5. Stats update on dashboard\n\n---\n\n## ⚠️ Important Limitations\n\n| Feature | Status |\n|---------|--------|\n| Live video streaming | ❌ Not available |\n| Real-time camera feed | ❌ Not available |\n| Continuous processing | ❌ Frame-based only |\n| Upload size | Max 10MB per request |\n| Processing time | 12 seconds max |\n\n**Why?** Vercel serverless functions are designed for fast, stateless requests - not 24/7 processes.\n\n---\n\n## 🔄 API Endpoints\n\n```\nGET  /                     → Dashboard\nGET  /login                → Login page\nPOST /login                → Authenticate\nGET  /logout               → Logout\n\nGET  /api/stats            → Get traffic state\nPOST /api/detect           → Detect vehicles in frame\nPOST /api/service/<lane>   → Toggle service mode\nPOST /api/accident         → Toggle emergency mode\nGET  /api/health           → Health check\n```\n\n---\n\n## 🐛 Troubleshooting\n\n### ❌ \"Deployment failed\"\n- Check `.gitignore` excludes `.pt` model files\n- Verify `public/` folder has `index.html`\n- Ensure `api/index.py` exists\n\n### ❌ \"Login not working\"\n- Verify environment variables in Vercel dashboard\n- Check browser console for errors\n\n### ❌ \"Frame processing timeout\"\n- Use smaller images (< 1MB)\n- Convert to JPEG format\n- Keep aspect ratio reasonable\n\n### ❌ \"404 on /api/detect\"\n- Ensure POST method is used\n- Check multipart form-data encoding\n- Verify `lane_id` parameter present\n\n---\n\n## 📊 Monitoring\n\nAfter deployment:\n\n1. **Vercel Dashboard** → Your Project\n2. **Analytics** tab → View traffic & errors\n3. **Logs** tab → Real-time function logs\n\n```bash\n# Via CLI\nvercel logs\n```\n\n---\n\n## 🔧 Environment Variables\n\nSet in **Vercel Dashboard → Settings → Environment Variables**:\n\n```\nADMIN_USER=admin\nADMIN_PASS=admin123\nSECRET_KEY=your-random-secret\n```\n\n---\n\n## 💡 Pro Tips\n\n1. **Faster detection** - Use smaller image resolution\n2. **Better accuracy** - Use clear, well-lit traffic images\n3. **Lower costs** - Few requests = minimal usage\n4. **Monitor logs** - Check Vercel Analytics dashboard\n5. **Update password** -
